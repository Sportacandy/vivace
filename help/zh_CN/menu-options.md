# 选项

**选项** 菜单包含首选项和界面配置。

- **首选项…**（`Ctrl+P`）— 主设置对话框：常规、界面、字幕、键盘和鼠标、播放列表、
  驱动器、电视、文件类型、更新、网络和高级。更改会立即应用；**取消** 会还原它们。
  - **网络** 包含 OpenSubtitles、YouTube、代理和投射四个选项卡；**代理** 用于
    配置可选的 HTTP 或 SOCKS5 代理，应用于整个程序（OpenSubtitles 搜索、更新
    检查，以及仅在 HTTP 时的媒体播放和 yt-dlp）；**投射** 用于固定
    *播放 ▸ 投射到 ▸ 智能手机/平板电脑* 所监听的端口。OpenSubtitles 账户密码
    和代理密码现在都安全存储在您操作系统的凭据管理器中，而不是存储在
    Vivace 自身的设置里。
- **在系统托盘中显示图标** — 让 Vivace 可从托盘访问。
- **工具栏**
  - **工具栏** / **控制栏** — 显示或隐藏每个栏。
  - **编辑主工具栏…** / **编辑控制栏…** — 选择显示哪些按钮及其顺序和图标大小。
- **状态栏**
  - **显示状态栏** 及其显示内容：**视频信息**、**音频信息**、**格式信息**、
    **比特率信息**、**帧计数器**、**显示总时长**、**显示剩余时间** 和
    **显示带毫秒的当前时间**。

**提示：** 整体布局（Basic / Mini / MPC）在 *首选项 ▸ 界面* 中选择。

## 导出 YouTube 下载所需的 Cookie

**Cookie 文件：** 字段（*首选项 ▸ 网络 ▸ YouTube*）可让 **下载并播放** 和
**外部工具** 这两种 YouTube 模式表现得如同您已登录 —— 这对于年龄限制、仅限
会员或其他需要账号验证的视频是必需的，也是解锁完整 HD/4K 下载的关键。它需
要一个采用经典 Netscape cookie 格式的纯文本 `cookies.txt` 文件（与 yt-dlp
自身的 `--cookies` 选项读取的格式相同）；Vivace 不会直接从浏览器的配置文件
中读取 Cookie。

**创建方法：**

1. 使用您想要使用其访问权限的账号，在您日常使用的浏览器中登录
   youtube.com。
2. 安装一个可导出 Netscape 格式的 Cookie 导出浏览器扩展 —— 对于
   Chrome、Edge 或 Brave，在浏览器的扩展商店中搜索类似
   “Get cookies.txt” 的扩展；对于 Firefox，搜索 “cookies.txt”。任何明确
   说明导出经典 Netscape `cookies.txt` 格式的扩展都可以使用。
3. 在打开 youtube.com 页面的标签页中，使用该扩展导出该网站的 Cookie，并
   将结果保存为磁盘上某处的 `.txt` 文件。
4. 在 Vivace 中，打开 *首选项 ▸ 网络 ▸ YouTube*，使用 **Cookie 文件：**
   旁边的 **浏览…** 按钮选择该文件。

**请注意：**

- `cookies.txt` 文件实际上等同于一个已保存的登录会话 —— 任何拥有该文件的
  人都可以冒充您的 YouTube 账号进行操作，直到 Cookie 过期或您退出登录为
  止。请将其存放在隐私安全的地方，不要与他人分享。
- Cookie 仅用于 **下载** 路径（下载并播放 / 外部工具）。Vivace 在
  **流式播放** 模式下有意从不发送 Cookie —— 已登录的流媒体 URL 会以
  Vivace 简单的视频播放器无法打开的方式绑定到该会话，因此即使配置了
  Cookie 文件，流式播放仍将保持匿名。
- Cookie 会过期。如果之前可以正常下载的内容开始失败，或回退到较低画质/
  公开版本的结果，请重新导出一个新的 `cookies.txt`。
