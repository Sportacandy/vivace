# 選項

**選項** 選單包含偏好設定與介面設定。

- **偏好設定…**（`Ctrl+P`）— 主設定對話方塊：一般、介面、字幕、鍵盤與滑鼠、播放清單、
  光碟機、電視、檔案類型、更新、網路與進階。變更會立即套用；**取消** 會還原它們。
  - **網路** 包含 OpenSubtitles、YouTube、代理和投影四個分頁；**代理** 用於
    設定可選的 HTTP 或 SOCKS5 代理，套用於整個應用程式（OpenSubtitles 搜尋、
    更新檢查，以及僅限 HTTP 時的媒體播放和 yt-dlp）；**投影** 用於固定
    *播放 ▸ 投影至 ▸ 智慧型手機／平板電腦* 所監聽的連接埠。OpenSubtitles
    帳戶密碼和代理密碼現在都安全地儲存在您作業系統的認證管理員中，
    而非儲存在 Vivace 自身的設定裡。
- **在系統匣中顯示圖示** — 讓 Vivace 可從系統匣存取。
- **工具列**
  - **工具列** / **控制列** — 顯示或隱藏每個列。
  - **編輯主工具列…** / **編輯控制列…** — 選擇顯示哪些按鈕及其順序與圖示大小。
- **狀態列**
  - **顯示狀態列** 及其顯示內容：**視訊資訊**、**音訊資訊**、**格式資訊**、
    **位元率資訊**、**畫格計數器**、**顯示總時長**、**顯示剩餘時間** 與
    **顯示含毫秒的目前時間**。

**提示：** 整體版面（Basic / Mini / MPC）在 *偏好設定 ▸ 介面* 中選擇。

## 匯出 YouTube 下載所需的 Cookie

**Cookie 檔案：** 欄位（*偏好設定 ▸ 網路 ▸ YouTube*）可讓 **下載並播放** 與
**外部工具** 這兩種 YouTube 模式表現得如同您已經登入 —— 這對於有年齡限
制、僅限會員或其他需要帳號驗證的影片是必要的，也是解鎖完整 HD/4K 下載的
關鍵。它需要一個採用傳統 Netscape cookie 格式的純文字 `cookies.txt` 檔案
（與 yt-dlp 自身的 `--cookies` 選項所讀取的格式相同）；Vivace 不會直接從
瀏覽器的設定檔中讀取 Cookie。

**建立方式：**

1. 在您平常使用的瀏覽器中登入 youtube.com，並使用您想要使用其存取權的
   帳號。
2. 安裝一個可匯出 Netscape 格式的 Cookie 匯出瀏覽器擴充功能 —— 若是
   Chrome、Edge 或 Brave，請在瀏覽器的擴充功能商店中搜尋類似
   「Get cookies.txt」的項目；若是 Firefox，請搜尋「cookies.txt」。任何
   明確標示會以傳統 Netscape `cookies.txt` 格式匯出的擴充功能皆可使用。
3. 在開啟 youtube.com 分頁的情況下，使用該擴充功能匯出該網站的 Cookie，
   並將結果儲存為磁碟上某處的 `.txt` 檔案。
4. 在 Vivace 中開啟 *偏好設定 ▸ 網路 ▸ YouTube*，並使用 **Cookie 檔案：**
   旁邊的 **瀏覽…** 按鈕選取該檔案。

**請注意：**

- `cookies.txt` 檔案實際上等同於一個已儲存的登入工作階段 —— 任何持有此
  檔案的人都能以您的 YouTube 帳號身分操作，直到 Cookie 過期或您登出為
  止。請將它存放在隱密的地方，切勿與他人分享。
- Cookie 僅用於 **下載** 路徑（下載並播放／外部工具）。Vivace 在
  **串流播放** 模式下刻意絕不傳送 Cookie —— 已登入的串流網址會以 Vivace
  這款單純的影片播放器無法開啟的方式與該工作階段綁定，因此即使設定了
  Cookie 檔案，串流播放仍會維持匿名。
- Cookie 有效期有限。若先前能正常下載的內容開始失敗，或退回為畫質較低／
  公開版本的結果，請重新匯出新的 `cookies.txt`。
